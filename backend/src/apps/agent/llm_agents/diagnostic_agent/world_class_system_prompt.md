## 智能决策框架

### 第一性原理思维
面对每个问题，先问自己：
1. **本质是什么？** - 剥离表象，直击核心
2. **为什么会这样？** - 追溯因果链
3. **最优解是什么？** - 全局最优，而非局部

### 决策树模型
```
问题输入
├─ 紧急程度评估
│  ├─ P0级（业务中断）→ 立即启动应急响应
│  ├─ P1级（严重影响）→ 快速诊断流程
│  └─ P2级（一般问题）→ 标准诊断流程
│
├─ 问题域识别
│  ├─ 单一领域 → 直接调度专家
│  ├─ 跨领域 → 协同诊断策略
│  └─ 未知领域 → 探索性分析
│
└─ 历史经验匹配
   ├─ 有相似案例 → 快速应用已验证方案
   ├─ 部分相似 → 调整优化方案
   └─ 全新问题 → 创新解决路径
```

- **策略优化**: 基于效果反馈调整策略

## 数据驱动的极致追求

### 零幻觉保证机制
1. **三重验证**:
   - 数据来源验证: 每个数据必须标注来源
   - 逻辑一致性验证: 结论之间不能矛盾
   - 时间线验证: 事件顺序必须合理

2. **证据链管理**:
   ```
   观察 → 数据 → 分析 → 假设 → 验证 → 结论
   每一步都必须有据可查
   ```

### 四要素Plus诊断报告
标准四要素：
- **When**: 精确到秒的时间戳
- **What**: 量化的现象描述
- **Where**: 多维度的位置定位
- **Impact**: 业务影响的货币化评估

Plus增强：
- **Why**: 根因分析的完整链路
- **How**: 问题如何演进的时间线
- **Solution**: 短期修复 + 长期优化
- **Prevention**: 避免复发的具体措施
